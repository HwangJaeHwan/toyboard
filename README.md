# 주요 기능

### 🏠 홈
- 게시판별 최신 게시글 5개 조회 및 메인 페이지에 표시
- 인기 게시글 리스트 제공
- 인기/최신 게시글은 Redis 캐싱을 활용하여 빠르게 응답

### 👤 회원가입 / 로그인
- 아이디, 닉네임 중복 여부 서버 검증
- 이메일 형식, 비밀번호 조건 등 유효성 검사 적용

### 📝 게시글 기능
- 페이징 기반 게시글 목록 조회
- 게시판 유형별(자유, QnA, 공지, 정보) 게시글 조회 및 정렬 지원
- 제목, 닉네임, 내용, 제목+내용으로 검색 가능 (Elasticsearch 연동)
- 추천 수, 조회 수 기준 정렬 지원
- `Summernote` 에디터 적용 → 이미지 포함 게시글 작성 가능
- 이미지 업로드 시 AWS S3 저장소 연동
- 추천 기능 → 1인 1회 제한으로 중복 방지

### 💬 댓글 기능
- 댓글 작성, 수정, 삭제 기능 제공
- 댓글 작성자 및 관리자만 수정/삭제 가능
- 댓글에 대해 up/down 기능 적용 (하나만 선택 가능)

### 📄 마이페이지 로그
- 게시글, 댓글, 추천, 신고 등 사용자 활동 기록
- 페이징 처리된 로그 목록 제공
- 각 로그 클릭 시 관련 게시글/댓글로 이동 가능


 
# 문제 해결

### 1. Redis 캐싱을 통한 조회 속도 최적화

**문제**  
홈페이지는 서비스 특성상 유저가 가장 많이 조회하는 페이지이기 때문에  
동일한 쿼리가 반복적으로 실행되면서 DB 부하와 응답 속도 저하가 발생할 수 있었습니다.

**해결**  
- Redis를 도입하여 카테고리별 최신 게시글 목록을 캐싱  
- 메인 페이지 요청 시 DB 대신 Redis에서 데이터를 응답하도록 변경    

**결과**  
- 평균 응답 속도 120ms → 3ms (약 40배 개선)
- DB 부하 및 커넥션 소모 감소
- 메인 페이지가 가장 많이 조회되는 상황에서도 안정적인 성능 유지


---

### 2. 스프링 프록시 AOP 내부 호출 문제 해결

**문제**  
같은 클래스 내에서 AOP가 적용된 메서드를 직접 호출하면 프록시 객체를 거치지 않아 AOP 기능이 동작하지 않는 문제가 발생했습니다.

**해결**  
- AOP 대상 메서드를 별도의 서비스 클래스로 분리하여 외부에서 호출

**결과**  
- AOP 기능이 정상 적용

---

### 3. 인덱스 및 페이징 최적화
**문제**  
게시글 목록 조회 시, 데이터가 많아질수록 조회시간이 오래걸림(10만건 이상)

**해결**  
- `게시글 타입 + 작성 시간` 복합 인덱스 적용  
- 인덱스를 활용하지 못하는 한방쿼리를 인덱스를 활용할 수 있는 쿼리들로 분해    

**결과**  
- 평균 조회 속도가 400ms에서 30ms로 약 10배 감소  
- 대규모 데이터셋에서도 일정한 성능 유지  
- 쿼리 가독성과 유지보수성 향상

---

### 4. Elasticsearch를 활용한 검색 성능 향상
**문제**  
MySQL `LIKE` 검색은 인덱스를 활용하지 못해 Full Scan이 발생했고,  
데이터가 많아질수록 성능 저하되는 문제가 있었습니다.  

**해결**  
- Elasticsearch + Nori Analyzer 적용  
- 하이라이트 적용해 사용자 경험 개선  

**결과**  
- 대규모 데이터셋에서도 실시간 수준의 검색 응답 속도 확보
- 검색 범위 확장: 제목 / 내용 / 닉네임 / 제목+내용 복합 검색 지원
- 사용자 입장에서 검색 정확도 + 가독성 모두 개선
